## 🤖 开发模式说明

### AI Agent 自动化开发

**核心理念：**
- 使用 AI Agent（如 Codex、Claude Code）自动生成代码
- 通过 `sessions_spawn` 并行执行任务
- 每个任务独立执行，自动报告结果
- 人类团队领导负责整体协调和决策

**特点：**
- ✅ 无需人工代码编写
- ✅ 自动生成测试用例
- ✅ 自动生成文档
- ✅ 24/7 自动执行
- ✅ 高效率（10-100x）

**时间估算：**
- 所有任务以 **10 分钟或 1 小时** 为单位
- 不再有"天"或"周"的概念
- 一个复杂的插件：10-30 分钟
- 一个完整的 Sprint：1-3 小时
- 整个项目：2-5 小时

## 🔄 工作流程

### 5 分钟自动化检查

**时间：** 每 5 分钟自动触发

**参与者：** 团队领导（AI Agent）

**工作流程：**
1. **检查任务状态**（1 分钟）
   - 查看所有运行中的任务
   - 检查超时任务（> 25 分钟）
   - 识别失败任务

2. **分配新任务**（2 分钟）
   - 自动识别可并行任务
   - 最多同时执行 3 个任务
   - 按技能和优先级分配

3. **进度报告**（1 分钟）
   - 只在异常或无进展时汇报
   - 正常有进展：不汇报
   - 异常或卡住：立即汇报

4. **清理完成任务**（1 分钟）
   - 清理已完成的任务
   - 更新任务状态
   - 记录执行历史

## 🏸 并行任务调度

### 并行策略

**核心原则：**
1. **依赖分析** - 只执行依赖已完成的任务
2. **资源限制** - 最多同时执行 3 个任务
3. **技能匹配** - 按技能和优先级分配
4. **独立执行** - 任务之间不互相干扰

**调度算法：**
```python
def schedule_tasks():
    """调度任务（AI Agent 自动化）"""
    while not all_completed():
        ready_tasks = []
        
        # 1. 找出所有可执行任务
        for task_id in TASKS.keys():
            can_run, reason = can_execute_task(task_id)
            if can_run:
                ready_tasks.append(task_id)
        
        # 2. 按优先级排序
        priority_order = {'P0': 0, 'P1': 1, 'P2': 2, 'P3': 3}
        ready_tasks.sort(key=lambda t: priority_order[TASKS[t]['priority']])
        
        # 3. 最多并行 3 个任务
        for task in ready_tasks[:3]:
            sessions_spawn(
                task=TASKS[task]['task_description'],
                agent_id=get_agent_id(task),
                label=task_id
            )
        
        # 4. 等待 5 分钟再检查
        time.sleep(300)
```

## 🎯 异常任务处理

### 异常处理策略（不跳过）

**核心原则：**
1. **重试机制** - 超时任务自动重试 3 次
2. **依赖分析** - 阻塞依赖当前任务的其他任务
3. **状态追踪** - 记录完整执行历史
4. **智能恢复** - 条件满足时自动恢复

**实现逻辑：**
```python
def handle_timeout_task(task_id):
    """处理超时任务（不跳过）"""
    retry_count = get_retry_count(task_id)
    
    # 1. 重试机制
    if retry_count < 3:
        delay = [60, 300, 600][retry_count]  # 1, 5, 10 分钟
        logger.warning(f"任务 {task_id} 重试中（第 {retry_count + 1}/3 次）")
        schedule_retry(task_id, delay)
        return True
    
    # 2. 重试 3 次都失败，阻塞任务
    logger.error(f"任务 {task_id} 重试 3 次均失败")
    
    # 3. 阻塞依赖任务
    dependents = get_dependents(task_id)
    if dependents:
        logger.error(f"以下任务被阻塞：{dependents}")
        for dep in dependents:
            mark_as_blocked(dep, reason=f"依赖任务 {task_id} 失败")
    
    # 4. 通知用户
    notify_user(f"任务 {task_id} 失败，被阻塞的任务：{dependents}")
    
    return False
```
