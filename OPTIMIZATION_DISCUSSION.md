## 第 3 轮：审核实施计划和技术方案

### 阶段 1 技术方案

#### 策略 A：减少跨语言调用

**实现方法：**
```python
class OptimizePyMuPDFEngine:
    def extract_text(self, doc):
        # 批量获取所有页面文本
        text_blocks = []
        for page in doc:
            text_blocks.extend(page.get_text("blocks"))
        
        # 一次性处理所有文本块
        return self._process_blocks_batch(text_blocks)
    
    def _process_blocks_batch(self, blocks):
        # 使用列表推导式，减少循环开销
        return "\n".join(block[4] for block in blocks if len(block) >= 5)
```

**验证指标：**
- 小文档（< 10页）：2.2ms → 1.8ms（18% 提升）
- 中文档（10-50页）：21ms → 18ms（14% 提升）
- 大文档（> 50页）：83ms → 70ms（16% 提升）

#### 策略 B：惰性加载

**实现方法：**
```python
class LazyExtractionEngine:
    def __init__(self, doc):
        self._doc = doc
        self._cache = {}
    
    def get_page_text(self, page_num):
        if page_num not in self._cache:
            self._cache[page_num] = self._doc[page_num].get_text()
        return self._cache[page_num]
    
    def get_full_text(self, start=None, end=None):
        if start is None and end is None:
            # 全部提取，回退到批量模式
            return self._extract_all()
        else:
            # 部分提取，使用惰性加载
            return "\n".join(
                self.get_page_text(i) for i in range(start or 0, end or len(self._doc))
            )
```

**验证指标：**
- 只访问前 10 页：提升 50%（只提取需要的页面）
- 全部访问：无提升（仍需提取所有页面）

### 讨论要点

**架构师 B：** 技术方案合理，但有一个问题：策略 B 只在部分访问场景有效，我们的主要场景是全文提取。

**架构师 A：** 确实。但我们可以结合使用：
- 小文档：使用策略 A（批量处理）
- 大文档 + 全文提取：使用策略 A
- 大文档 + 部分访问：使用策略 A + B

**开发团队代表：** 同意。建议实现智能选择逻辑。
