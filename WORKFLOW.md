## 🎯 测试驱动开发（TDD）工作流程

### 步骤 1：编写测试用例（Red）

**参与者：** 测试工程师

**任务：**
```python
# 1. 理解需求
# 需求：插件应该支持配置更新

# 2. 编写测试用例
def test_plugin_config_update():
    """测试插件配置更新"""
    manager = PluginManager()
    plugin = MockPlugin()
    
    # 设置初始配置
    old_config = {"key1": "value1"}
    manager.set_plugin_config("mock", old_config)
    
    # 更新配置
    new_config = {"key1": "value2", "key2": "value2"}
    manager.set_plugin_config("mock", new_config)
    
    # 验证配置已更新
    config = manager.get_plugin_config("mock")
    assert config == new_config
    
    # 验证钩子被调用
    assert plugin.on_config_update_called

# 3. 运行测试（应该失败）
pytest test_plugin_config_update
# 结果：❌ AssertionError: plugin.on_config_update_called
```

---

### 步骤 2：编写最小实现（Green）

**参与者：** Python 开发者

**任务：**
```python
# 1. 实现最小功能
class BasePlugin:
    def on_config_update(self, old_config, new_config):
        self._config = new_config
        # 添加钩子调用标记
        self.on_config_update_called = True

# 2. 运行测试（应该通过）
pytest test_plugin_config_update
# 结果：✅ PASS
```

---

### 步骤 3：重构优化（Refactor）

**参与者：** Python 开发者 + 系统架构师

**任务：**
```python
# 1. 优化代码结构
class BasePlugin:
    def __init__(self):
        self._config = None
        self._logger = logging.getLogger(f"plugin.{self.name}")
        self._hooks = {}  # 添加 Hook 系统
    
    def on_config_update(self, old_config, new_config):
        self._config = new_config
        self._trigger_hook("config_update", old_config, new_config)
    
    def _trigger_hook(self, event, *args, **kwargs):
        """触发 Hook"""
        if event in self._hooks:
            for callback in self._hooks[event]:
                callback(*args, **kwargs)

# 2. 运行测试（确保仍然通过）
pytest test_plugin_config_update
# 结果：✅ PASS + 更好的代码结构
```
