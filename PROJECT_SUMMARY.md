- **问题：** Gateway cron tool 超时（60 秒）
- **原因：** 配置 schema 不支持 cron 配置项
- **解决：** 切换到 systemEvent
- **教训：** 使用文件配置更简单

### 2. 5 分钟检查间隔 ✅
- **原因：** AI Agent 效率高（2-10 分钟）
- **效果：** 实时监控、快速响应

### 3. 异常任务处理 ⚠️（重要！）
- **问题：** 有些任务执行超时但无完成消息
- **当前解决：** 跳过异常任务，继续其他任务

## ⚠️ 异常任务处理改进建议

### 当前策略（⚠️ 不安全）

```python
if task_time > 25_minutes and no_completion_message:
    skip_task()  # ⚠️ 简单跳过
```

**潜在风险：**
1. **依赖任务可能失败** - 跳过的任务可能是其他任务的前置条件
2. **数据不一致** - 跳过的任务可能生成必要文件
3. **测试覆盖不完整** - 跳过的测试任务可能导致质量下降
4. **文档缺失** - 跳过的文档任务可能导致用户困惑

### 推荐解决方案

**混合方案：依赖分析 + 重试机制 + 状态追踪**

**核心策略：**
1. **依赖分析** - 检查任务依赖关系，只跳过无依赖的任务
2. **重试机制** - 超时任务自动重试 3 次（1 分钟、5 分钟、10 分钟）
3. **状态追踪** - 记录完整执行历史，支持诊断和恢复
4. **自动恢复** - 如果条件满足，自动恢复阻塞的任务

**实现逻辑：**
```python
def handle_timeout_task(task_id):
    """处理超时任务（安全版）"""
    retry_count = get_retry_count(task_id)
    
    # 1. 重试机制
    if retry_count < 3:
        logger.warning(f"任务 {task_id} 重试中（第 {retry_count + 1}/3 次）")
        schedule_retry(task_id)
        return
    
    # 2. 重试次数用完，检查是否可跳过
    if can_skip_task(task_id):
        # 检查依赖任务
        dependents = get_dependents(task_id)
        
        if dependents:
            logger.error(f"任务 {task_id} 有依赖任务，不能跳过：{dependents}")
            mark_as_blocked(task_id)
        else:
            logger.warning(f"任务 {task_id} 标记为跳过（无依赖）")
            mark_as_skipped(task_id)
    else:
        logger.error(f"任务 {task_id} 不能跳过，必须完成")
        mark_as_blocked(task_id)
```
